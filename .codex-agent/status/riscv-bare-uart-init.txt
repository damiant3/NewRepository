summary=UART output works without init but receive (LSR[0] polling) never sees data. x86-64 __start inits COM1 (LCR=0x03, FCR=0xC7, MCR=0x0B, IER=0x01 baud=115200). RISC-V __start has no UART init. Add NS16550A init sequence to EmitStart for BareMetal target before calling main. Blocks bare metal serial input and self-compile.
