title=ARM64 missing CCE encoding — strings are raw ASCII
status=active
updated=2026-03-29
agent=linux
summary=ARM64 backend has no CCE at all. EmitTextLit uses UTF8.GetBytes instead of CceTable.Encode. Print path dumps raw bytes with no CCE→Unicode conversion. All char builtins (is-digit, is-whitespace, is-letter) return wrong results because they check CCE ranges against ASCII values. Fix: port CCE-native encoding to ARM64 (EmitTextLit, AddRodataString, EmitCharLit, lookup tables, print conversion, I/O boundaries). Same work x86-64 and RISC-V already have.
